Implementation of Eifel Retransmission Timer in CoAP

Students : Vishal Rathod, Sanjana Krishnam, Ayush Kumar, Gauri Baraskar .
Mentor : Mohit P. Tahiliani


Introduction


Eifel Retransmission Timer suggests an alternative way of RTO estimation in TCP. It uses a RTT based RTO prediction which overcomes the shortcomings of the default TCP RTO estimation. TCP Eifel is made up of three parts out of which we focus on the Eifel Retransmission Timer.
In this project we have implemented this retransmission timer for Constrained Application Protocol(CoAP). CoAP uses a fixed RTO value with BEB in case of retransmissions, after implementing Eifel Retransmission Timer in CoAP, the RTO values will be set based on network state determined from the RTT values. In our work we have demonstrated that this RTO estimation leads to better network performance across various metrics such as throughput and latency.

Steps to Reproduce the results

  1. Clone the Contiki OS repository : Link
  2. Download the Changefiles using the link given in this page
  3. Extract and paste the changefiles in apps/er-coap directory within Contiki OS repository.
